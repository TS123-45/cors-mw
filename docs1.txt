Configuring CORS Options
You can configure multiple CORS settings.

app.use(cors({
  origin: "http://127.0.0.1:5500",
  methods: ["GET", "POST", "PUT", "DELETE"],
  allowedHeaders: ["Content-Type", "Authorization"]
}));

This controls:
Option

Meaning
origin
allowed frontend
methods
allowed HTTP methods
allowedHeaders
allowed headers

6️⃣ Dynamic Origin
Sometimes you want to check origins programmatically.
Example:
const allowedOrigins = [
  "http://127.0.0.1:5500",
  "http://localhost:5500"
];

app.use(cors({
  origin: function(origin, callback) {

    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error("CORS not allowed"));
    }

  }
}));
Meaning:

If origin is trusted → allow
If origin not trusted → reject
Used in many production APIs.
